# 项目规划（RL-Agent-Trading）

## 目标
在8-10周内形成一个可复现、可解释、可面试防守的RL-Agent交易研究项目。

## 里程碑
1. M1（第1周）：Quick Start跑通，产出基线模型与评估结果。
2. M2（第3周）：最小交易环境与回测流程跑通。
3. M3（第5周）：Agent编排流程上线（train/eval/report）。
4. M4（第8周）：多seed与walk-forward结果整理成研究报告。

## 最小可交付
- 可运行代码库（含README、配置、脚本、日志）
- 至少2种算法对比（DQN + PPO）
- 至少2类基线对比（B&H + MA）
- 一份项目报告（方法、结果、局限）

## 风险与缓解
- 风险：训练不稳定
  - 缓解：固定seed，多次运行，缩小动作空间
- 风险：回测结果过拟合
  - 缓解：walk-forward、样本外评估
- 风险：项目过大推进慢
  - 缓解：先保最小闭环，再逐步加复杂度
